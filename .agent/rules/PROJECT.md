# Контекст для агента (YouTube Reklama Skip)

## Стек и версии
- **Язык**: Чистый JavaScript (Vanilla JS, ECMAScript 2022+)
- **Стили**: Чистый CSS3 (Flexbox/Grid, CSS Variables, Animations)
- **Браузерное API**: Chrome Extension API (Manifest V3 - Service Workers, Promises)

## Соглашения по коду
### Naming
- Файлы: snake_case или kebab-case (напр. `background.js`, `popup.html`).
- Переменные/Функции: `camelCase`.
- Классы/DOM Идентификаторы: Для JS классов — `PascalCase`, для CSS и ID — `kebab-case` (напр. `#btn-save`, `.settings-panel`).

### Структура
- В начале каждого файла должны быть собраны константы (включая селекторы DOM).
- Разделять логику работы с DOM и бизнес-логику (насколько это возможно без фреймворка).
- Использовать `console.debug()` или `console.log()` с префиксом `[RSKIP]` для удобства дебаггинга.

## Особенности проекта
- Проект **НЕ ДОЛЖЕН** использовать бандлеры (Webpack/Vite/Rollup) ради максимального упрощения архитектуры, так как мы хотим нулевую нагрузку на систему сборки и быстрый старт расширения напрямую в браузере.
- Код должен быть оптимизирован. Например, в `youtube.js` обработчик события `timeupdate` (встроенный в HTML5) срабатывает очень часто. Внутри него должна быть только легковесная проверка (`if (currentTime >= skipStart && currentTime <= skipEnd)`).
- Скрипт Gemini (`gemini.js`) крайне хрупок: Google в любой день может изменить DOM у `gemini.google.com`. Все селекторы (кнопка отправки, поле ввода, блоки с ответами) нужно вынести в легко изменяемый конфиг внутри `gemini.js`.
